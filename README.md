# Ultimate Smart Home

Проект «Ultimate Smart Home» — модульная демонстрация архитектуры умного дома на Python с асинхронными сервисами.

Коротко
- Асинхронный дизайн на `asyncio` для сервисов и управления устройствами.
- Чёткая модульная структура: `core`, `devices`, `services`, `utils`.
- Лёгкая расширяемость: добавление новых устройств и сервисов без правки ядра.

Ключевые возможности
- Управление виртуальными устройствами и их интеграция со службами.
- Асинхронная обработка событий и задач.
- Структура, готовая для тестов и дальнейшей интеграции в реальные API.

Структура репозитория (основное)
- `main.py` — пример запуска (асинхронный вход в приложение).
- `requirements.txt` — зависимости проекта.
- `core/`, `devices/`, `services/`, `utils/` — основные пакеты проекта.

Требования
- Python 3.8+
- Зависимости в `requirements.txt` (рекомендуется виртуальное окружение).

Быстрый старт
1. Создайте виртуальное окружение и активируйте его.

Windows (PowerShell):
```powershell
python -m venv .venv
. .venv\Scripts\Activate.ps1
```

Linux / macOS:
```bash
python3 -m venv .venv
source .venv/bin/activate
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Запустите пример:
```bash
python main.py
```

Фразы для разработчика
- Добавляйте новые драйверы устройств в `devices/`.
- Реализуйте бизнес-логику в `services/`.
- Общие утилиты и помощники — в `utils/`.

Тесты и импорт-проверки
- В репозитории может быть `test_imports.py` — запуск его проверяет корректность импортов и структуры пакетов.

Разработка
- Используйте типизацию и юнит-тесты для новых модулей.
- Следуйте паттерну разделения ответственности между `core` и `services`.

Контакты и вклад
